# Junior Growth Marketing Specialist Portfolio Setup

## Overview

This repository was created as the first step of the portfolio project for the Junior Growth Marketing Specialist application process.

The purpose of this step was to set up the required tools, create a GitHub repository, document the process, and submit the README.md file as requested.

## Tools Installed

The following tools were installed and prepared:

- Cursor IDE
- Claude Code extension in Cursor
- Codex extension in Cursor
- GitHub account
- Public GitHub repository

## Steps Completed

1. Installed Cursor IDE.
2. Created a public GitHub repository.
3. Opened a local project folder in Cursor.
4. Installed the Claude Code extension in Cursor.
5. Installed the Codex extension in Cursor.
6. Created a README.md file to document the setup process.
7. Prepared the project for upload to GitHub.
8. Uploaded the README.md file to the GitHub repository.

## Issues Encountered and How I Solved Them

The main challenge was setting up a clean workflow across several tools and making sure the process was easy to document and review. Since this task involved Cursor, GitHub, Claude Code, and Codex, I focused on validating each step before moving forward.

I solved this by keeping the project structure simple, confirming that the required tools were installed, and documenting the setup process in a way that would be easy for someone else to follow. I treated the README not just as a task requirement, but as a small communication exercise: clear context, completed steps, blockers, and final status.

This reminded me that growth work often depends on the same habits: move quickly, but verify the setup; test the workflow before scaling it; and document clearly so the next step becomes easier.

## What I Learned

This task helped me become more familiar with Cursor, GitHub, and AI-assisted workflow tools. I also learned the importance of documenting a process clearly, especially when working with new tools.

As a Junior Growth Marketing Specialist, I believe this type of workflow is relevant because modern growth work often involves testing tools, setting up systems, documenting experiments, and communicating progress clearly.
